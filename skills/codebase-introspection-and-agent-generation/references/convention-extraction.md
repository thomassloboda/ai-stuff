# Convention Extraction Reference

## Goal

Infer local conventions that agents should obey in this repository.

## Examples of conventions to extract

- folder structure and module boundaries
- file naming conventions
- test naming and placement
- barrel-file usage
- commit or PR conventions
- dependency direction rules
- framework-specific patterns
- error handling conventions
- configuration management patterns

## Rules

- Conventions should be extracted from repeated patterns, not one isolated file.
- If multiple conventions coexist, note that the repository may be transitional.
- Prefer documenting the dominant convention plus the notable exceptions.
